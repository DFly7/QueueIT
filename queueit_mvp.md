# QueueIT - Sunday Demo Requirements (MVP)
**Deadline:** Sunday, March 15th, 2026

## Goal
A functional web/mobile demo showing a cross-platform (Apple & Spotify) room where a song added to the queue plays in sync for both users.

## Core Requirements

### 1. Cross-Platform Auth
- [ ] **Apple Music:** OAuth flow/token generation.
- [ ] **Spotify:** OAuth flow/token generation.
- [ ] **Storage:** Link user session to their music provider.

### 2. Room & Queue Management
- [ ] **Room Creation:** Host can create a room (unique ID/URL).
- [ ] **Unified Search:** Search interface that queries *either* Apple or Spotify and displays results.
- [ ] **The Vote Queue:** Users can add tracks to a shared list; upvoting moves a song to the top of the next-to-play.

### 3. "Host-Sync" Engine (The "Bridge")
- [ ] **Metadata Normalization:** Store songs in the queue by Title/Artist/ISRC (not just platform ID).
- [ ] **Status Sync:** 
    - Host (Apple) plays a song -> Backend updates Room state with `current_song` and `timestamp`.
    - Guest (Spotify) client polls Room state -> Searches for the normalized metadata on Spotify -> Triggers `play` at the correct offset.

### 4. Minimal UI
- [ ] Basic "Join Room" screen.
- [ ] Active queue display with vote buttons.
- [ ] Player controls (visible for Host, sync-only for Guest).
